# Alzheimer's Disease Drug Repurposing

Uses the DRKG knowledge graph and pre-trained TransE embeddings to rank DrugBank compounds by their predicted treatment score for Alzheimer's disease.

## Setup

```bash
pip install pandas numpy torch requests
```

On first run, the script downloads DRKG (~200MB) automatically.

## Usage

```bash
python alzheimers_drug_repurposing.py
```

This writes `alzheimers_top100_predictions.tsv` and prints ranked results to stdout. Known AD drugs that appear in the top 100 are flagged as a sanity check.

## Output

`alzheimers_top100_predictions.tsv` has four columns: `rank`, `drug_id` (e.g. `Compound::DB00843`), `drug_name` (DrugBank ID), and `score`. Scores are always negative due to logsigmoid — closer to 0 means stronger prediction.

## Scoring

TransE L2 scoring function: `d = γ - ||h + r - t||₂`, `score = logsigmoid(d)`

Two treatment relations are used: `Hetionet::CtD::Compound:Disease` and `GNBR::T::Compound:Disease`.

## Neo4j (optional)

To explore the knowledge graph visually:

1. Start Neo4j and create a database
2. Copy `alzheimers_top100_predictions.tsv` and `drkg_with_headers.tsv` into the Neo4j import folder
3. Run `neo4j_import_predictions.cypher`

`drkg_with_headers.tsv` is generated by the script automatically.

## Files

| File | Description |
|------|-------------|
| `alzheimers_drug_repurposing.py` | Main script |
| `utils.py` | Downloads and extracts DRKG data |
| `alzheimers_known_drugs.tsv` | Known AD drugs for validation |
| `neo4j_import_predictions.cypher` | Neo4j import and query scripts |

## Data

- DRKG: 97,238 entities, 5.8M relationships from DrugBank, Hetionet, GNBR, String, IntAct, DGIdb
- Embeddings: pre-trained TransE L2 from [gnn4dr/DRKG](https://github.com/gnn4dr/DRKG)

## Citation

- DRKG: Ioannidis VN et al. (2020)
- TransE: Bordes A et al. (2013)
